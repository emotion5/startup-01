# 2025-12-11 업데이트: 데이터 구조 및 컴포넌트 리팩토링 요약

이 업데이트는 애플리케이션의 핵심 데이터 모델을 '스타트업' 중심에서 '프로젝트' 중심으로 전환하고, 관련 컴포넌트들을 이에 맞춰 리팩토링하는 데 중점을 두었습니다. 주요 변경 사항과 데이터 구조 개선 내용은 다음과 같습니다.

## 1. 데이터 구조 재구성 (`src/data/startups.ts` -> `src/data/mockData.ts`)

*   기존의 `startups.ts` 파일이 삭제되고, `src/data/mockData.ts` 파일이 새로 생성되었습니다.
*   `Startup` 인터페이스가 `Project` 및 `Company` 두 개의 독립적인 인터페이스로 분리되었습니다.
    *   `Company` 인터페이스: 기업의 기본 정보(이름, 설명, 설립일, 주소, 웹사이트, 이메일)를 정의합니다.
    *   `Project` 인터페이스: 각 프로젝트의 세부 정보(제목, 설명, `companyId`를 통한 기업 참조, 공개 여부(`visibility`), 진행 상태(`status`), 진행률(`progress`), 3D 모델 타입, 색상)를 정의합니다.
*   데이터는 이제 `companies` 배열과 `projects` 배열로 분리되어 관리됩니다. 이로써 기업과 프로젝트 간의 1:N 관계를 명확하게 표현할 수 있게 되었습니다.

## 2. 상태 관리 로직 업데이트 (`src/hooks/useStore.ts`)

*   Zustand 스토어가 새로운 `Project` 및 `Company` 데이터 모델을 지원하도록 업데이트되었습니다.
*   `selectedStartup` 상태는 `selectedProject`와 `selectedCompany`로 분리되어, 각 프로젝트와 관련 기업 정보를 독립적으로 관리합니다.
*   `setSelectedStartup` 함수는 `setSelectedProject`와 `setSelectedCompany`로 대체되었습니다.
*   `openModal` 함수는 이제 `modalContent`를 선택적으로 받으며, 주로 `selectedProject`와 `selectedCompany` 정보를 기반으로 모달 내용을 구성합니다.

## 3. UI 컴포넌트 개선 및 데이터 바인딩 변경

*   **`src/components/canvas/Props/InfoKiosk.tsx`:**
    *   `startup` prop 대신 `project`와 `company` prop을 받도록 변경되었습니다.
    *   클릭 이벤트 시 `setSelectedProject` 및 `setSelectedCompany`를 호출하여 상태를 업데이트합니다.
    *   키오스크의 색상 및 표시 이름이 `project.color`와 `project.title`을 따르도록 수정되었습니다.
*   **`src/components/ui/Dashboard/StartupCard.tsx` (삭제) -> `src/components/ui/Dashboard/ProjectCard.tsx` (신규):**
    *   기존 `StartupCard` 컴포넌트는 삭제되었고, `Project` 데이터를 표시하기 위한 `ProjectCard` 컴포넌트가 새로 추가되었습니다.
    *   `ProjectCard`는 프로젝트의 제목, 관련 기업 이름, 진행 상태, 진행률 등의 정보를 시각적으로 보여줍니다.
*   **`src/components/ui/Overlay/InfoModal.tsx`:**
    *   모달 내부에서 `selectedStartup` 대신 `selectedProject`와 `selectedCompany`의 정보를 사용하여 상세 내용을 표시하도록 로직이 변경되었습니다.
    *   프로젝트 진행 상태, 진행률 바, 관련 기업 정보(설립일, 주소, 웹사이트, 이메일) 등이 새롭게 추가되거나 개선되었습니다.
*   **`src/pages/DashboardPage.tsx`:**
    *   `startups` 데이터 대신 `projects`와 `companies` 데이터를 사용하도록 변경되었습니다.
    *   `ProjectCard` 컴포넌트를 사용하여 공개(`public`) 상태의 프로젝트 목록을 렌더링하며, 각 프로젝트에 대한 기업 정보를 연결하여 표시합니다.
*   **`src/pages/ExhibitionPage.tsx`:**
    *   URL 파라미터 `id`를 기반으로 `project`와 `company`를 조회하고, 이를 `InfoKiosk` 컴포넌트에 전달합니다.
    *   선택된 프로젝트와 기업 정보를 `setSelectedProject`와 `setSelectedCompany`를 통해 전역 상태에 저장합니다.

## 4. 스타일링 및 UI/UX 개선

*   `src/index.css` 및 `src/components/ui/Overlay/InfoModal.module.css`에서 모달의 배경색, 텍스트 색상, 테두리, 그림자, 최대 높이 및 오버플로우 처리 등 전반적인 스타일링이 개선되어 사용자 경험이 향상되었습니다.

이러한 변경 사항들을 통해 애플리케이션은 더욱 유연하고 확장 가능한 데이터 구조를 갖추게 되었으며, 사용자에게는 개별 프로젝트에 대한 더욱 풍부하고 체계적인 정보를 제공할 수 있게 되었습니다.
